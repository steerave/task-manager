# Daily Note Redesign — Design Spec

**Date:** 2026-04-02
**Status:** Draft
**Scope:** Restructure daily note layout, add auto-regeneration on task mutations, add calendar caching, add `task priority` command

---

## Problem

The current daily note organizes tasks by time bucket (Today / This Week / Next Week). This hides tasks that aren't due soon but require significant effort to complete. The user needs to see all open tasks at a glance to make informed prioritization decisions. Additionally, the daily note only updates on explicit `task today` runs, so it goes stale as tasks are added and completed throughout the day.

## Goals

1. Show all open tasks in one view, grouped by domain, sorted by priority then due date
2. Auto-regenerate the daily note on any task mutation (add, done, update, delete, waiting, priority)
3. Cache calendar events so auto-regeneration doesn't make unnecessary iCloud API calls
4. Add a `task priority` shortcut command

## Non-Goals

- Focus/pinning system (removed from scope)
- Obsidian plugin for live updates on direct .md edits
- Changes to task file format or ID system

## Daily Note Layout

```markdown
## Today — April 2, 2026

### All Open Tasks

#### Personal
- [ ] ⚠️ Pay Invoice 23425 — ProTech *(due Mar 31 — overdue)* · High
- [ ] ⚠️ Check wexhealth.com payment *(due Apr 1 — overdue)* · Medium
- [ ] Apple River Mortgage updates *(due Apr 2)* · Medium
- [ ] Get hair cut *(due Apr 3)* · Medium
- [ ] MIT Complete module 1 *(due Apr 7)* · Medium
- [ ] Some ongoing task · Low

#### Personal Projects
- [ ] Brainstorm granola AI meeting notes system *(due Apr 3)* · Medium
- [ ] Build real-time interview aide tool *(due Apr 8)* · Medium

### Waiting On
- [/] CPAP follow up with Kelly Home Medical *(due Apr 3)* — Personal

### Events
- 13:00-14:00 · Call with coach Rachel
- NO SCHOOL FOR KIDS *(all day)*

### Completed Today
- [x] Wex review and application form completion

---
*Generated by task-manager · 2026-04-02 14:38*
```

### Sort Order (within each domain)

1. **Priority:** High > Medium > Low
2. **Due date ascending:** soonest first within same priority
3. **No due date:** bottom of their priority group

### Visual Rules

- Overdue tasks get `⚠️` prefix and `— overdue` label after the date
- Waiting tasks use `[/]` checkbox (existing Obsidian convention)
- Completed tasks use `[x]` checkbox
- Priority shown inline after domain: `· High`, `· Medium`, `· Low`
- Sections with no items are omitted entirely

### Removed

- This Week / Next Week sections — replaced by the flat All Open Tasks view
- Needs Triage section — inbox tasks appear in All Open Tasks like any other open task
- Due Today / Overdue as separate sections — overdue tasks sort to the top naturally and get the `⚠️` marker

## Note Refresh Layer

New module: `src/core/noteRefresher.ts`

### Interface

```typescript
interface RefreshOptions {
  freshCalendar: boolean  // true = fetch from iCloud + update cache
                          // false = read from cache file
}

function refreshDailyNote(config: Config, options: RefreshOptions): Promise<void>
```

### Flow

1. Scan all tasks from vault (always fresh read from .md files)
2. Get calendar events:
   - `freshCalendar: true` → call `fetchTodayEvents()`, write result to cache file
   - `freshCalendar: false` → read cache file; if date doesn't match today, return empty array
3. Call `generateDailyNote(tasks, events)` to produce markdown
4. Write the daily note file to `{vault}/DailyNotes/YYYYMMDD - Daily Task.md`

### Calendar Cache

File: `{vault}/.calendar-cache.json`

```json
{
  "date": "2026-04-02",
  "events": [
    {
      "uid": "abc123",
      "name": "Call with coach Rachel",
      "date": "2026-04-02",
      "startTime": "13:00",
      "endTime": "14:00",
      "isAllDay": false
    }
  ]
}
```

- Cache is valid for the current day only
- If `date` field doesn't match today, cache is stale — return empty events
- Cache file is gitignored (lives in vault, not repo)

## Command Changes

### Modified Commands

Each command calls `refreshDailyNote(config, { freshCalendar: false })` after completing its primary operation:

| Command | Trigger Point |
|---------|--------------|
| `task add` | After writing task file |
| `task done` | After marking task done |
| `task update` | After saving updates |
| `task delete` | After removing task file |
| `task waiting` | After setting waiting status |

### Modified: `task today`

1. Run checkbox sync (parse existing note for checked items, mark tasks done)
2. Call `refreshDailyNote(config, { freshCalendar: true })`

This replaces the current inline note generation logic. `task calendar today` remains as an alias for `task today` (both do the same thing now).

### New: `task priority`

```bash
task priority <id> <high|medium|low>
```

- Resolves shorthand ID (reuses existing pattern from done/update/delete)
- Removes old `priority/*` tags from the task
- Adds new `priority/<level>` tag
- Calls `refreshDailyNote(config, { freshCalendar: false })`
- Prints confirmation: `Priority set: <task name> → High`

### `dailyNoteGenerator.ts` Changes

- Pure function signature stays: `generateDailyNote(tasks: Task[], events?: CalendarEvent[]): string`
- New layout replaces old: All Open Tasks (by domain, priority+date sort) → Waiting On → Events → Completed Today
- Remove This Week / Next Week / Due Today / Overdue / Needs Triage sections
- Add `⚠️` overdue prefix logic
- Add priority-first sort within domain groups
- Handle no-due-date tasks at bottom of priority group
